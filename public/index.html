How It Works

Three steps. No local relay dance.

The flow stays short: install the extension, connect your agent, then claim one tab.

01

Load the extension

The extension keeps the browser side connected and exposes your private route.

02

Copy the MCP

Use your browser-specific MCP URL so the agent targets your browser only.

03

Claim a tab

Each task gets its own tab group and a cleaner handoff for browser work.

Features

Built for clean cBrowse agent-browser handoff.

Live bridge status

See whether the connected browser is ready before the agent starts acting.

Dark by default

Starts in dark mode and supports a light mode toggle when you want the page brighter.

Skill + llms.txt

Use the hosted MCP directly or attach the cBrowse skill when your client supports it.

Get Started

Install once. Copy once. Start work.

The setup is just extension, MCP, and an optional cBrowse skill layer.

/01 INSTALL

Load the unpacked extension and confirm the bridge is connected.

/02 CONNECT

Add the browser-specific MCP to your agent. Use a stable `sessionId` and short `taskName` when you want cleaner tab groups.

/03 OPTIONAL SKILL

Attach the hosted cBrowse skill when your client supports extra browser instructions.
